插入排序

  1. 直接插入排序

    算法思想：首先以一个元素为有序的序列，然后将后面的元素依次插入到有序的序列中合适的位置直到所有元素都插入有序序列。

    实现代码

    ​x
    void InsertSort(ElemType A[],int n){
    
  
    
  
        int i,j;
    
  
        for(i=2;i<=n;i++)   
    
  
            if(A[i].key<A[i-1].key){    
    
  
                A[0]=A[i];          //复制为哨兵，A[0]不存放元素，作为辅助变量使用
    
  
                for(j=i-1;A[0].key<A[j].key;--j)
    
  
                    A[j+1]=A[j];    //所有比待插入元素值大的都往后移一位，腾出空位
    
  
                A[j+1]=A[0];        //插入 
    
  
            }
    
  
    }
    
  
    ​

    算法模拟

    空间复杂度：O(1) 就地算法，辅助单元为哨兵Array[0]

    时间复杂度：

    最好情况：表中元素为正序，Tn=O(n), 比较次数 : n, 不需要移动；

    最坏情况：表中元素为逆序，Tn=O(n²), 比较次数 : ∑i, i∈[2,n], 移动次数: ∑i+1, i∈[2,n];

    平均情况：元素随机，Tn=O(n²), 比较次数=移动次数=n²/4

    稳定性：稳定

    适用性：顺序存储、链式存储的线性表

    特点：每次边比较然后边移动元素，直到找到插入的位置。有可能直至最后一趟排序所有元素都不在其最终位置上。

  2. 折半插入排序

    算法思想：将比较和移动这两个操作分离出来，也就是先利用折半查找找到插入的位置，然后一次性移动元素，再插入该元素。

    实质：直接插入算法的改进

    实现代码

    xxxxxxxxxx
    
    
    
    
    
    
    
    
    
      
    
        
    
        
    
        
    void InsertSort(ElemType A[],int n){
    
      
    
      
        int i,j,low,high,mid;
    
      
        for(i=2;i<=n;i++){       //i是待插入的元素下标，A[0]~A[i-1]为有序数列  
    
      
            A[0]=A[i];          //保存待插入的值           
    
      
            low=1;high=i-1; 
    
      
            while(low<=high){   //折半查找  
    
      
                mid=(low+high)/2;   
    
      
                if(A[mid].key>A[0].key) high=mid-1; 
    
      
                else low=mid+1; 
    
      
            }                  //找到了待插入的位置 接下来从后往前依次后移元素腾出位置
    
      
            for(j=i-1;j>=high+1;--j)A[j+1]=A[j];        
    
      
            A[high+1]=A[0];     //插入，high指向有序数列的末位      
    
      
        }
    
      
    }

    算法模拟

    空间复杂度：O(1) 就地算法，辅助单元为哨兵Array[0]

    时间复杂度：

    最好情况：表中元素为正序，Tn=O(nlogn), 比较次数 = nlogn, 不需要移动；

    最坏情况：表中元素为逆序，Tn=O(n²), 比较次数 = nlogn , 移动次数= ∑i+1, i∈[2,n];

    平均情况：元素随机，Tn=O(n²), 比较次数= nlogn, 移动次数=n²/4

    稳定性：稳定

    适用性：顺序存储的线性表

    特点：先查后移，比较次数与表的初始状态无关

     

  1. 希尔排序

    算法思想：希尔排序本质上还是插入排序，只不过是把待排序序列分成几个子序列，再分别对这几个子序列进行直接插入排序。

    实质：直接插入算法的改进

    实现代码

    xxxxxxxxxx
    
      
    
      
    
      
    
      
    
        
    
          
    
          
    
          
    /* A[i].key是待插入的关键字，i-dk之前的都是有序的，如果待插入的比有序序列最后一个小，则需要进行排序(进入if语句块)，如果大则不需要(跳出if语句块)。s待插入关键字之前以dk为增量的关键字只要比待插入关键字大的都往后移动dk位
    
        
    
        
    */
    
        
    void ShellSort (ElemType A[],int n){
    
        
                   int i,j;
    
        
        for(dk=n/2;dk>=1;dk=dk/2)       //初始增量为总长度的一半，之后依次除2且向下取整直至d=1
    
        
            for(i=dk+1;i<=n;++i)
    
        
                if(A[i].key<A[i-dk].key){
    
        
                    A[0]=A[i];          //待插入关键字暂存在A[0]
    
        
                    for(j=i-dk;j>0 && A[0].key< A[j].key; j-=dk)
    
        
                        A[j+dk]=A[j];       
    
        
                    A[j+dk]=A[0];       //找到了待插入的位置，就将待插入关键字插入这个位置
    
        
                }
    
        
    }

    算法模拟

    空间复杂度：O(1) 就地算法，辅助单元为常数个辅助单元

    时间复杂度：依赖增量序列函数，涉及到数学上暂未解决难题

    稳定性：不稳定

    适用性：顺序存储的线性表

    特点：希尔排序的每一轮都会使整个序列变得越来越有序，最后一轮当增量为1的时候，整个序列几乎都是有序的，所以进行直接插入排序会提高排序的效率。

     

交换排序

  1. 冒泡排序

    算法思想：假设待排序的表长为n, 从后往前（或从前往后）相邻元素之间两两比较，较小者（或较大者）往上浮，直至序列全部比较完，完成一趟冒泡。

    实现代码

    xxxxxxxxxx
    
      
    
      
    
      
    
      
    
        
    
          
    
          
    
          
    void BubbleSort(ElemType A[],int n){
    
        
    
        
        for(i=0;i<n-1;i++){
    
        
            bool flag=false;                //当整个序列都有序的时候，标志位不变 
    
        
            for(j=n-1;j>i;j--)              //一趟冒泡过程，每趟进行n-1次比较
    
        
                if(A[j-1].key>A[j].key){    //如果前面的元素比后面的大，则需要做交换
    
        
                    ElemType temp=A[j-1].key;
    
        
                      A[j-1].key=A[j].key;
    
        
                      A[j].key=temp;    
    
        
                    flag=true;             //发生了数据交换 修改标志位
    
        
                }
    
        
            if(flag==false)return ;         //本趟遍历后没有发生交换，排序完成
    
        
        }
    
        
    }

  2. 快速排序

    算法思想：将问题划分为若干个子问题，分别求解（分治法）。将待排序的序列按选定的枢轴值划分为两个子序列，枢轴值左侧序列元素均小于(均大于)枢轴值，右侧序列元素均大于(均小于)枢轴值。枢轴值则处于最终位置。该过程为一趟快速排序。而后对其两个子序列分别递归进行快速排序，以完成最终排序。

    实质：冒泡的改进

    实现代码

    xxxxxxxxxx
    
      
    
      
    
      
    
      
    
        
    
          
    
          
    
          
    /*划分函数——pivot < A[high] ?high-- : ( A[low]<->A[high],low++) */
    
        
    
        
    int Partition(ElemType A[],int low,int high){  //low是待排序的序列起始下标，high是末尾下标
    
        
        ElemType pivot=A[low];                   //通常使用第一个元素作为枢轴
    
        
        while(low<high){        
    
        
            while(low<high&&A[high]>=pivot) --high;//先从末尾往前找到第一个比枢轴小的元素
    
        
            A[low]=A[high];                      //用high的元素替换low的元素
    
        
            while(low<high&&A[low]<=pivot) ++low;  //再从开头往后找到第一个比枢轴大的元素
    
        
            A[high]=A[low];                      //用low的元素替换high的元素
    
        
        }
    
        
        A[low]=pivot;           //枢轴元素存放到最终位置
    
        
        return  low;            //返回存放枢轴的最终位置
    
        
    }
    
        
    ​
    
        
    /*递归法，快速排序*/
    
        
    void QuickSort(ElemType A[],int low,int high){
    
        
        if(low<high){                    //low和high值要合法         
    
        
            int pivotpos=Partition(A,low,high); 
    
        
            QuickSort(A,low,pivotpos-1);  //分治递归左半部分    
    
        
            QuickSort(A,pivotpos+1,high); //分治递归右半部分
    
        
        }
    
        
    }

    算法模拟

    空间复杂度：

    平均情况: O(log n )；

    最好情况：log (n+1)向上取整

    最坏情况：进行n-1次递归 O(n)栈深度为n-1；

    时间复杂度：与划分子序列是否对称有关

    平均情况：O(nlog n);

    最好情况：O(nlog n);

    最差情况：两个区域分别包含n-1个元素和0个元素——序列表基本有序时，O(n²)

    稳定性：不稳定

    适用性：顺序存储、链表的线性表

    特点：平均性能最优的排序算法，且在排序过程中并不产生有序子序列，每趟排序后有一个元素将会位于最终位置。

    注：递归复杂度的表达式：T(n) = aT(n/b) + f(n) ；其中n/b是子问题的大小，比如快排每次划分子问题之后的大小，a是子问题大小的数量，f(n)是分解问题和合并问题所需要的时间

选择排序

  1. 简单选择排序

    算法思想：每次从待排序序列中选择一个最小(最大)的元素，作为有序子序列的第I个元素。需进行n-1趟，且比较次数以及排序趟数与序列初始排序无关。先记录min值下标最后再进行交换。

    实现代码

    xxxxxxxxxx
    
      
    
      
    
      
    
      
    
        
    
          
    
          
    
          
    void SelectSort(ElemType A[],int n){
    
        
    
        
        for(i=0;i<n-1;i++){                //最后一个元素不需要排序
    
        
               min=i;                       //min存的是当前最小元素所在下标，初值设为第i个
    
        
               for(j=i+1;j<n;j++)           //从第i个元素往后找，一直要找到最后一个元素
    
        
               if(A[j]<A[min]) min=j;       //更新min值
    
        
               if(min!=i) {                 //如果A[i]!=A[min]，则进行交换
    
        
                     ElemType temp=A[i];
    
        
                     A[i]=A[min];
    
        
                     A[min]=temp;
    
        
                }   
    
        
        }
    
        
    }

    最好情况：O(nlog n);

    特点：平均性能最优的排序算法，且在排序过程中并不产生有序子序列，每趟排序后有一个元素将会位于最终位置。

    注：递归复杂度的表达式：T(n) = aT(n/b) + f(n) ；其中n/b是子问题的大小，比如快排每次划分子问题之后的大小，a是子问题大小的数量，f(n)是分解问题和合并问题所需要的时间

  2. 堆排序

    算法思想：将无序序列建成一个堆，得到关键字最小（或最大）的记录；输出堆顶的最小（大）值后，使剩余的n-1个元素重又建成一个堆，则可得到n个元素的次小值；重复执行，得到一个有序序列，这个过程叫堆排序。

    实现代码

    xxxxxxxxxx

      开始检查，直到根结点(注意根结点下标不是0，是从1开始存储)*/
    
        
    /*建堆*/
    
        
    void BuildMaxHeap(ElemType A[],int len){
    
        
        for(int i=len/2;i>0;i--)
    
        
            AdjustDown(A,i,len); 
    
        
    }

    /*下沉法*/
    
        
    void AdjustDown(ElemType A[],int k,int len){ 
    
        
        //A是存储堆的数组,k是需要检查的结点下标,len是堆中结点个数
    
        
        A[0]=A[k];                  //A[0]暂存这个需要检查的结点值
    
        
        for(i=2*k;i<=len;i*=2){     //从这个结点的左孩子开始往下比较，
    
        
                                 // 如果发生交换，对交换过的结点继续和它的孩子比较
    
        
            if(i<len&&A[i]<A[i+1])
    
        
             i++;         //如果右孩子大一些，就只要考虑和右孩子比较
    
        
            if(A[0]>=A[i])  
    
        
             break;       //如果这个结点的值不小于它的较大孩子结点值 则不需要交换
    
        
            else{ 
    
        
            A[k]=A[i];   //如果这个结点的值小于它的较大孩子
    
        
                              //结点值则将较大的孩子结点值赋值给该结点 
    
        
            k=i;        //i赋值给k也就是从i开始继续往下检查 直到所有结点检查结束 
    
        
            }
    
        
    }
    
        
        A[k]=A[0];          //检查到最后k的值就是最后一轮交换过的结点位置下标 将该结点换过去      
    
        
    }
    
        
    /*上浮*/
    
        
       void AdjustUp(ElemType A[], int k){
    
        
        A[0]=A[k];
    
        
           int i=k/2;       //若结点值大于双亲，则双亲下调并继续向上比较
    
        
        while(i>0&&A[i]<A[0]){
    
        
               A[k]=A[i];   //双亲下调
    
        
            k=i;
    
        
               i=k/2;       //向上比较
    
        
        }
    
        
           A[k]=A[0];       //赋值    
    
        
    }

归并排序

  1. 归并排序

    算法思想：待排序表含有n个记录，则可以看成是n个有序的子表，每个子表长度为1，然后两两归并，得到 ⌈n/2⌉个长度为2或1的有序表；再两两归并，直到合并成一个长度为n的有序表为止，这种方法称为2-路归并。分治思想

    实现代码

    xxxxxxxxxx
    
      
    
      
    
      
    
      
    
        
    
          
    
          
    
          
    ElemType *B=(ElemType *)malloc((n+1)*sizeof(ElemType)); //辅助数组B(动态分配内存)
    
        
    
        
    ​
    
        
    /*表A的两段A[low…mid]和A[mid+1…high]各自有序，将它们合并成一个有序表*/
    
        
    void Merge(ElemType A[],int low,int mid,int high){
    
        
        for(int k=low;k<=high;k++)B[k]=A[k];            //将A中所有元素复制到B中
    
        
        for(int i=low,j=mid+1,k=i;i<=mid&&j<=high;k++){  //k是归并之后数组的下标计数器
    
        
            if(B[i]<=B[j])                            //比较B的左右两段中的元素
    
        
                A[k]=B[i++];                          //将较小值复制到A中
    
        
            else
    
        
                A[k]=B[j++];
    
        
        }
    
        
        while(i<=mid)   A[k++]=B[i++];  //若第一个表未检测完，直接将剩下的部分复制过来
    
        
        while(j<=high)  A[k++]=B[j++];  //同上，复制剩余值，且二者只运行其一
    
        
    }   
    
        
    /*排序*/
    
        
    void MergeSort(ElemType A[],int low,int high){
    
        
        if(low<high){
    
        
            int mid=(low+high)/2;       //从中间划分两个子序列
    
        
            MergeSort(A,low,mid);       //对左侧子序列进行递归排序
    
        
            MergeSort(A,mid+1,high);    //对右侧子序列进行递归排序
    
        
            Merge(A,low,mid,high);      //归并
    
        
        }
    
        
    }

    算法模拟

    空间复杂度：

    平均情况: O(n)；

    时间复杂度：每趟时间为O(n)，共需log n趟

    平均情况：O(nlog n);

    最好情况：O(nlog n);

    最差情况：O(nlog n);

    稳定性：稳定

    适用性：顺序存储、链表的线性表

计数排序

  1. 计数排序

    算法思想： 计数排序的核心在于将输入的数据值转化为键存储在额外开辟的数组空间中。作为一种线性时间复杂度的排序，计数排序要求输入的数据必须是有确定范围的整数。

    实现代码

    xxxxxxxxxx

    void CountSort(vector<int> &arr, int maxVal) {
    
        
        int len = arr.size();
    
        
        if (len < 1)
    
        
            return;
    
        
        vector<int> count(maxVal+1, 0);     //计数数组
    
        
        vector<int> tmp(arr);
    
        
        for (auto x : arr)
    
        
            count[x]++;
    
        
        for (int i = 1; i <= maxVal; ++i)
    
        
            count[i] += count[i - 1];
    
        
        for (int i = len - 1; i >= 0; --i) {
    
        
            arr[count[tmp[i]] - 1] = tmp[i];
    
        
            count[tmp[i]]--;                //注意这里要减1
    
        
        }
    
        
    }

算法模拟

空间复杂度：

平均情况: O(n)；

时间复杂度：O(n)，不需要进行比较

稳定性：稳定

适用性：顺序存储、链表的线性表

  1. 基数排序

    算法思想：基数排序(也叫桶排序)是一种很特别的排序方法，它不是基于比较进行排序的，而是采用多关键字排序思想（即基于关键字各位的大小进行排序的），借助“分配”和“收集”两种操作对单逻辑关键字进行排序。基数排序又分为最高位优先（MSD）排序和最低位优先（LSD）排序。

    实现代码（了解即可）

    xxxxxxxxxx
    
      
    
      
    
      
    
      
    
        
    
          
    
          
    
          
    /*
    
        
    
        
    *求数据的最大位数,决定排序次数
    
        
    */
    
        
    int maxbit(int data[], int n) 
    
        
    {
    
        
        int d = 1; //保存最大的位数
    
        
        int p = 10;
    
        
        for(int i = 0; i < n; ++i)
    
        
        {
    
        
            while(data[i] >= p)
    
        
            {
    
        
                p *= 10;
    
        
                ++d;
    
        
            }
    
        
        }
    
        
        return d;
    
        
    }
    
        
    void radixsort(int data[], int n) //基数排序
    
        
    {
    
        
        int d = maxbit(data, n);
    
        
        int tmp[n];
    
        
        int count[10]; //计数器
    
        
        int i, j, k;
    
        
        int radix = 1;
    
        
        for(i = 1; i <= d; i++) //进行d次排序
    
        
        {
    
        
            for(j = 0; j < 10; j++)
    
        
                count[j] = 0; //每次分配前清空计数器
    
        
            for(j = 0; j < n; j++)
    
        
            {
    
        
                k = (data[j] / radix) % 10; //统计每个桶中的记录数
    
        
                count[k]++;
    
        
            }
    
        
            for(j = 1; j < 10; j++)
    
        
                count[j] = count[j - 1] + count[j]; //将tmp中的位置依次分配给每个桶
    
        
            for(j = n - 1; j >= 0; j--) //将所有桶中记录依次收集到tmp中
    
        
            {
    
        
                k = (data[j] / radix) % 10;
    
        
                tmp[count[k] - 1] = data[j];
    
        
                count[k]--;
    
        
            }
    
        
            for(j = 0; j < n; j++) //将临时数组的内容复制到data中
    
        
                data[j] = tmp[j];
    
        
            radix = radix * 10;
    
        
        }
    
        
    }

    算法模拟

    空间复杂度：需要开辟关键字基的个数个队列，所以空间复杂度为O(r)

    时间复杂度：一次"分配"和一次"收集"时间复杂度为O(n+r)。d次就需要O(d(n+r))的时间复杂度

    稳定性：稳定

    适用性：顺序存储、链表的线性表
